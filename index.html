HarnessX

Compose, adapt, and evolve the behavior around your agent.

HarnessX makes the runtime harness a typed, serializable interface between model and environment. Reusable processors shape behavior, traces guide evolution, and the resulting trajectories become training signal.

Overview

The harness matters as much as the model.

Keep model selection separate from behavior. A HarnessConfig describes context, tools, memory, control, evaluation, safety, observability, and training export in one composable object.

01 / Compose

Change behavior without rewriting the agent.

Attach, replace, or remove typed processors at lifecycle hooks.

02 / Adapt

Search the harness from execution traces.

AEGIS proposes scoped edits and gates them with auditable checks.

03 / Evolve

Turn successful runs into better training data.

Reward-annotated trajectories connect runtime improvement to model learning.

Architecture

One core, many agent behaviors.

A single run loop hosts interchangeable processor bundles, adaptive search, and training-ready trajectories.

Method

Trace-driven improvement, made explicit.

Harness configurations behave like states, typed edits like actions, and verifier scores like feedback. Deterministic gates decide what ships.

from harnessx import BaseTask, HarnessConfig
from harnessx.core.model_config import ModelConfig
from harnessx.providers.anthropic_provider import AnthropicProvider

model = ModelConfig(main=AnthropicProvider("claude-sonnet-4-6"))
harness = model.agentic(HarnessConfig())
result = await harness.run(BaseTask(description="What is 2 + 2?"))
