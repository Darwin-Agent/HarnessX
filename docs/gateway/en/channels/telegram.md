# Telegram Channel Configuration Guide

---

## Prerequisites

You need a Telegram account and access to **BotFather** (Telegram's official bot management tool) to create a bot.

---

## Step 1: Create a Bot

1. Search for `@BotFather` in Telegram and start a conversation
2. Send `/newbot`
3. Enter the bot's **display name** (e.g. `HarnessX Assistant`)
4. Enter the bot's **username** (must end with `bot`, e.g. `harnessx_bot`)
5. BotFather will return a **Bot Token** in the format:
   ```
   123456789:ABC-DEF1234ghIkl-zyx57W2v1u123ew11
   ```
   Keep this safe and do not expose it.

---

## Step 2: Configure Bot Permissions (Optional)

Send the following commands to BotFather to configure the bot:

```
/setprivacy — Set privacy mode
  Choose DISABLED to let the bot receive all messages in groups (not just /commands)
  Choose ENABLED (default) to receive only /commands and @mention messages

/setjoingroups — Allow or disallow the bot from being added to groups
/setcommands — Set the bot command list (shown at the bottom of the input box)
```

Recommended command list setup:

```
/setcommands
help - Show help information
reset - Clear conversation history
cancel - Cancel the current task
```

---

## Step 3: Get User ID (Optional, for Allowlist)

To restrict the bot to specific users only:

1. Search for `@userinfobot` in Telegram
2. Send any message and it will return your **user ID** (a plain number)
3. Add this ID to the `allowed_users` list

---

## Step 4: Add the Bot to a Group (Optional)

1. In Telegram group settings → **Add Members** → search for your bot's username
2. After adding, confirm the bot has **Send Messages** permission
3. If privacy mode is ENABLED, users must @mention the bot in messages to trigger it (or set `require_mention: false` and disable privacy mode)

## Config File

### Direct Message Mode (Minimal Configuration)

```yaml
channels:
  telegram:
    enabled: true
    channel_type: telegram
    bot_token: "123456789:ABC-DEF1234ghIkl-zyx57W2v1u123ew11"
```

### Personal Assistant with Allowlist

```yaml
channels:
  telegram:
    enabled: true
    bot_token: "123456789:ABC-DEF1234..."
    allowed_users:
      - "123456789"    # Your Telegram user_id
    max_steps: 20
```

### Group Bot

```yaml
channels:
  telegram:
    enabled: true
    bot_token: "123456789:ABC-DEF1234..."
    require_mention: false   # No @mention required in groups (privacy mode must be disabled)
    reply_in_thread: true    # Reply in thread (reduces flooding)
    session_mode: per_user   # Independent session per user
```

## Complete Field Reference

| Field | Type | Required | Default | Description |
|-------|------|----------|---------|-------------|
| `bot_token` | string | ✅ | — | Bot Token generated by BotFather |
| `allowed_users` | list | | `[]` (everyone) | Allowlist of user_ids permitted to use the bot; empty = everyone |
| `require_mention` | bool | | `false` | Whether @mention is required in groups to trigger the bot |
| `reply_in_thread` | bool | | `false` | Whether to reply within a Thread |
| `webhook_secret_token` | string | | — | Secret token for Webhook mode (not needed in Polling mode) |
| `session_mode` | string | | `shared` | `shared` or `per_user` |
| `max_steps` | int | | `20` | Maximum steps per task |
| `token_budget` | int | | `100000` | Token limit per task |
| `workspace` | string | | `auto` | File workspace mode |

## Connection Mode

Gateway uses **Long Polling** mode to receive messages. No public IP or Webhook configuration is required, making it suitable for local and intranet deployment.

> Telegram Bot API Long Polling fetches up to 100 updates per poll. Gateway continuously polls to ensure no messages are lost.

## Troubleshooting

### Bot does not respond to messages

1. Confirm the `bot_token` is correct (including the full part after `:`)
2. For group messages: check the bot's privacy mode; in ENABLED state, only /commands and @mentions are received
3. View logs: `hx-gateway logs -n 100 | grep telegram`

### Message send failure (429 Too Many Requests)

Telegram has strict rate limits:
- Maximum 30 messages per second (global)
- Maximum 20 messages per minute to the same chat

Gateway has a built-in Token Bucket rate limiter that handles this automatically. If rate limits are still triggered, increase `edit_interval` appropriately.

### MarkdownV2 formatting errors (special characters in messages)

Telegram MarkdownV2 requires escaping `_*[]()~>#+-=|{}.!` and other characters. Gateway's formatter escapes these automatically. If issues arise, check whether the Agent output contains unescaped special characters.

### `allowed_users` not working in groups

`allowed_users` uses `sender_id` (a numeric string). Ensure you are entering the user's numeric ID, not their username.

## Complete Configuration Example

```yaml
channels:
  telegram:
    enabled: true
    channel_type: telegram
    bot_token: "123456789:ABC-DEF1234ghIkl-zyx57W2v1u123ew11"
    allowed_users:
      - "123456789"
      - "987654321"
    require_mention: false
    reply_in_thread: false
    session_mode: per_user
    max_steps: 20
    token_budget: 80000
```
